# The Holomatic Principle — Roy Ascott

## Author's Definition

Ascott names the principle in the title essay of *Telematic Embrace* (ch. 20, "Is There Love in the Telematic Embrace?", 1990) and gives it verbatim: "Each individual computer interface is an aspect of a telematic unity, such that to be in or at any one interface is to be in the virtual presence of all the other interfaces throughout the network of which it is a part. This might be defined as the 'holomatic' principle in networking." (`Directly stated`, Source 1)

The definition is in two clauses. The first is the claim itself — each interface is an *aspect of a telematic unity*, so to be at one node is to be virtually present at all nodes. The second is its justification, which Ascott makes informational rather than metaphorical: it holds "because all the data flowing through any access node of the network are equally and at the same time held in the memory of that network: they can be accessed, through cable or satellite links, from any part of the planet at any time" (`Directly stated`, Source 1). The coinage is a contraction of *holo-* (the holographic part-contains-the-whole structure) and *-matic* (the automatic, machinic networking it describes): a network in which each node holds the whole because the whole network's memory is held simultaneously at every node.

By *The Architecture of Cyberception* (ch. 27, 1994) the term has stabilized as Ascott's standard formula, restated compactly: "The holomatic principle is that each individual interface to the net is an aspect of a telematic unity: to be in or at any one interface is to be in the virtual presence of all the other interfaces throughout the network." (`Directly stated`, Source 2) There it becomes the perceptual content of [[Cyberception]] — "Cyberception gives us access to the holomatic media of the cybernet" — and the principle names not just a network property but a faculty for apprehending it.

## Context

Ascott introduces the holomatic principle as the technical underwriting of his larger telematic-art programme. Once content is held simultaneously across every node, the [[The Interface as Doorway|interface]] ceases to be a window onto a finished work and becomes "a doorway to undecidability," a site where content is created rather than received and then "reinserted, transformed by the process of interaction, back into the network" for transformation at the interfaces of other users elsewhere (`Directly stated`, ch. 20). The principle is therefore the structural basis of [[Network Consciousness]] and of [[Telematic Art]]'s distributed, participatory model: a single shared planetary data-memory in which presence is decoupled from physical location and synchronous time — users "do not need to be in the same place at the same time."

The figure is prefigured two chapters earlier in *Gesamtdatenwerk* (ch. 18) as "a holographic unity" of the interface — same part-as-whole claim before the coinage is settled — and Ascott demonstrated it concretely at the 1986 Venice Biennale "Planetary Network," where "the data flow at any one point was the data flow at all other points in the system" (`Directly stated`, ch. 23). The companion concept note [[The Holomatic Principle]] carries the fuller exposition, the variants across chapters, and the questions the principle raises; this note isolates the explicit definitional statement.

## Differences from Other Uses

- **vs the everyday network sense of "access."** Ordinary network terminology treats a node as a *terminal* that requests data stored elsewhere (client/server). Ascott's holomatic claim is stronger and more idealized: the whole network's data are "equally and at the same time held in the memory of that network," so the node does not reach out to a remote store — it *is* the store, an aspect of the telematic unity rather than a discrete endpoint (`Directly stated`).

- **vs McLuhan's [[The Global Village]].** McLuhan's electric simultaneity collapses distance and makes the planet a single acoustic field of involvement, but his paradigm case is *broadcast* — one-to-many. Ascott extends simultaneity from broadcast toward a literal shared, two-way data-memory in which every node holds the whole and inserts back into it; the holomatic principle is the global village reconceived as participatory network memory rather than transmission (`Interpretive synthesis`).

- **vs Hayles's [[Virtuality]].** Hayles theorizes virtuality as the cultural perception that material objects are interpenetrated by information patterns. Ascott's "virtual presence of all the other interfaces" is a narrower, affirmative use: presence itself — not just objects — is recoded as a virtual, networked co-presence, embraced rather than diagnosed (`Interpretive synthesis`).

- **vs the broader [[Material vs Immaterial]] tension.** The holomatic principle is Ascott's most enthusiastically immaterialist move: presence, memory, and the artwork are relocated from the material object and physical place into the simultaneously shared data-memory of the network. It is the same distributed, ubiquitous, surveillance-capable condition that later vault authors (Han's [[Digital Panopticon]]) read as domination rather than embrace — making the holomatic principle a key Ascott anchor for that eight-author tension (`Interpretive synthesis`).
